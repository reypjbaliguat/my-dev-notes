<SectionCard title="Two categories of Git">

- Centralized Version Control Systems (CVCS) - if server goes offline, you cannot collaborate with others
- Distributed Version Control Systems (DVCS) - this is commonly used today

</SectionCard>

<SectionCard title="Why Git?">

- Free
- Open Source
- Super Fast
- Efficient handling of large projects / Scalable
- Strong support community
- More than 90% of open source projects use Git

</SectionCard>

<SectionCard title="Using Git">

- Command Line - fastest and easiest way to use Git
- VSCode - source control panel - extension is Git Lens
- GUI Clients - graphical user interfaces for Git (e.g., GitHub Desktop, Sourcetree, Git Kraken)

</SectionCard>

<SectionCard title="Why Command Line?">

- GUI tools have limitations
- GUI tools are not always available

</SectionCard>

<SectionCard title="Installing Git">

- Press `Command + Space` and type terminal in macOS
- Search `cmd` in Windows
- Type `git --version` to check if Git is installed
- BASH means `Bourne Again SHell`

</SectionCard>

<SectionCard title="Settings">

- Name - `git config --global user.name "Your Name"`
- Email - `git config --global user.email`
- Default text editor - `git config --global core.editor "code --wait"`
- Line Ending - `git config --global core.autocrlf true` for Windows, `input` for macOS/Linux
- Set git values in VSCode - `git config --global -e`
- Core autocrlf(Carriage Return Line Feed) - handle line endings - `git config --global core.autocrlf true` if windows or `input` for macOS/Linux
- To get more info about a setting - `git config --help`
- To get short summary of settings - `git config -h`

</SectionCard>
